# 练习 22：SCAN — 基于游标的迭代

## 你将学到
- 使用 `SCAN` 安全迭代键
- 使用 `HSCAN` 迭代哈希字段
- 使用 `SSCAN` 迭代集合成员
- 使用 `ZSCAN` 迭代有序集合成员
- 理解为什么 SCAN 是生产安全的而 KEYS 不是

## 为什么重要

`KEYS *` 在扫描时阻塞整个 Redis 服务器——在有数百万键的数据库中，这可能导致数秒的暂停。`SCAN` 使用基于游标的方法，返回小批次且永不阻塞。这是生产环境中迭代键的唯一安全方式。

## 常见陷阱

- **COUNT 只是提示**：Redis 可能返回比请求更多或更少的键。
- **可能出现重复**：迭代期间添加的键可能出现两次。
- **游标是不透明的**：不要试图解释游标值——只需传递给下一次调用。
