# 练习 07：发布/订阅

## 你将学到

- 使用 `PUBLISH` 向频道发布消息
- 使用 `SUBSCRIBE` 订阅频道并实时接收消息
- 使用 `PSUBSCRIBE` 进行基于模式的订阅
- 理解 Pub/Sub 消息传递语义（即发即忘）

## 为什么重要

Redis Pub/Sub 实现了应用组件之间的实时消息传递。它是聊天系统、实时通知、服务健康监控和事件驱动架构的基础。与消息队列（见练习 08）不同，Pub/Sub 是即发即忘的——消息不会持久化，订阅者只能收到其主动订阅期间发布的消息。

## 核心概念

### 发布-订阅模型

```
发布者 ──PUBLISH channel "hello"──→ Redis ──→ 订阅者 1（已订阅 "channel"）
                                            ──→ 订阅者 2（已订阅 "channel"）
                                            ✗  订阅者 3（尚未订阅——错过消息）
```

## 常见陷阱

- **消息不持久化**：Pub/Sub 不是消息队列。使用 Streams 进行持久化消息传递。
- **SUBSCRIBE 阻塞连接**：在 redis-py 中，`pubsub.listen()` 会阻塞并产生消息。使用单独的连接或线程进行订阅。
- **模式订阅匹配所有频道**：`PSUBSCRIBE *` 会订阅所有内容——谨慎使用。
- **断开前取消订阅**：使用 `UNSUBSCRIBE` 清理订阅，避免残留的订阅者条目。
