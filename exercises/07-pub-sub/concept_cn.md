# 练习 07：发布/订阅

## 你将学到
- 使用 `PUBLISH` 向频道发布消息
- 使用 `SUBSCRIBE` 订阅频道并实时接收消息
- 使用 `PSUBSCRIBE` 进行基于模式的订阅
- 理解即发即忘的传递语义

## 为什么重要
Redis Pub/Sub 实现了应用组件之间的实时消息传递——聊天系统、实时通知、服务健康监控。与 Streams（Ex08）不同，Pub/Sub 是即发即忘的，消息不持久化。

## 常见陷阱
- 消息不持久化——Pub/Sub 不是消息队列。使用 Streams 实现持久化。
- SUBSCRIBE 阻塞连接——使用单独的 PubSub 对象和监听线程。
- 模式订阅匹配所有匹配的频道——`PSUBSCRIBE *` 会订阅所有内容。
