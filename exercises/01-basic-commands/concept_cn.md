# 练习 01：基础命令

## 你将学到

- 使用 `PING` 连接 Redis 并验证连接
- 使用 `SET` 和 `GET` 存储和读取字符串值
- 使用 `DEL` 删除键
- 使用 `EXISTS` 检查键是否存在
- 使用 `EXPIRE` 设置过期时间，用 `TTL` 查看剩余时间
- 理解 `KEYS *` 是 O(N) 操作——仅用于调试，生产环境禁用
- 使用 `TYPE` 查看键的类型

## 为什么重要

这是 Redis 最基本的 CRUD 操作。`SET`、`GET`、`DEL`、`EXISTS` 和 `EXPIRE` 是所有后续练习的基础。在深入哈希、列表、流等数据结构之前，必须先掌握键的生命周期：创建、修改、过期、删除。

## 核心概念

### Redis 键空间

Redis 是一个键值存储。所有数据都存储在一个键下。键是二进制安全的字符串（实践中我们使用可读的字符串）。键空间是扁平的——没有像关系数据库那样的"表"或"集合"。约定使用冒号分隔的命名空间：`user:1`、`article:42:title`、`session:abc123`。

### 键的生命周期

```
SET user:1 "Alice"     →  键存在，值为 "Alice"
EXPIRE user:1 3600     →  键存在，TTL 从 3600 秒开始倒计时
TTL user:1             →  返回剩余秒数（如 3587）
DEL user:1             →  键立即删除
TTL user:1             →  返回 -2（键不存在）
```

### TTL 返回值

| TTL 值 | 含义 |
|--------|------|
| 正整数 | 键存在，将在 N 秒后过期 |
| `-1` | 键存在但未设置过期时间（持久化） |
| `-2` | 键不存在 |

### KEYS 是 O(N) 操作

`KEYS pattern` 会扫描整个键空间。在有数百万键的数据库中，这会阻塞 Redis 服务器。生产环境请使用 `SCAN`（后续练习会涉及）。`KEYS` 仅适用于学习和调试。

## 你将实践

1. 连接 Redis 并发送 `PING`（应返回 `PONG`）
2. 使用 `SET name "Alice"` 设置键，用 `GET name` 读取
3. 设置多个键并用 `EXISTS` 验证
4. 设置带过期时间的键，用 `TTL` 观察倒计时
5. 删除键并验证其已消失（`TTL` 返回 `-2`）
6. 使用 `KEYS *` 列出所有键，用 `TYPE` 查看类型

## 常见陷阱

- **KEYS 阻塞服务器**：`KEYS` 是同步 O(N) 扫描，绝不要在生产代码中使用，请用 `SCAN` 替代。
- **TTL 返回 -1 与 -2**：`-1` 表示"未设置过期"（键是持久的），`-2` 表示"键不存在"。不要混淆。
- **EXPIRE 覆盖**：对已有 TTL 的键调用 `EXPIRE` 会将计时器重置为新值。
- **SET 覆盖过期时间**：不带 `EX`/`PX` 的 `SET` 会删除任何已有的 TTL——键变为持久化。使用 `SET key value EX 3600` 一步设置值和过期时间。
- **DEL 静默执行**：`DEL` 返回删除的键数（单个键为 0 或 1）。键不存在时不会报错。
