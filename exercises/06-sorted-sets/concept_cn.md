# 练习 06：有序集合

## 你将学到

- 使用 `ZADD` 添加带分数的成员
- 使用 `ZRANGE` 和 `ZREVRANGE` 按排名获取
- 使用 `ZRANK` 和 `ZREVRANK` 获取排名
- 使用 `ZSCORE` 获取分数
- 使用 `ZCOUNT` 按分数范围计数
- 使用 `ZINCRBY` 递增分数
- 使用 `ZREM` 删除成员

## 为什么重要

有序集合是 Redis 最通用的数据结构。每个成员都有一个分数（浮点数），成员始终按分数排序。可用于排行榜、优先队列、限流窗口、时间序列数据以及任何需要有序数据和快速范围查询的问题。

## 核心概念

### 排行榜模式

```
ZADD game:leaderboard 9500 "alice" 8200 "bob" 10100 "dave"
ZREVRANGE game:leaderboard 0 2 WITHSCORES
# 返回: [("dave", 10100), ("alice", 9500), ("bob", 8200)]
ZRANK game:leaderboard "alice"  → 1（从零开始，即第二名）
```

## 常见陷阱

- **相同分数 → 字典序排列**：两个分数为 100 的成员不会具有相同的排名——它们按字母顺序排列。
- **ZRANK 从零开始**：最高排名成员（分数最低）的排名为 0。
- **ZADD 会更新已有成员**：添加已存在的成员会更新其分数。
- **大型有序集合**：内部结构是跳表 + 哈希表。操作复杂度为 O(log N)。
