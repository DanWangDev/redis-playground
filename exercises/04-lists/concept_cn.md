# 练习 04：列表

## 你将学到

- 使用 `RPUSH` + `LPOP` 构建队列（FIFO）
- 使用 `LPUSH` + `LPOP` 构建栈（LIFO）
- 使用 `LRANGE` 检查范围
- 使用 `LTRIM` 限制集合大小
- 使用 `LLEN` 获取长度

## 为什么重要

Redis 列表是针对头/尾操作优化的链表（O(1)）。它们非常适合队列、栈、动态流和任务系统。阻塞弹出操作使工作池无需忙等待轮询。

## 常见陷阱

- **LRANGE 是 O(S+N)**：起始偏移量越大越慢。在巨大列表上使用 `LRANGE list 0 -1` 代价高昂。
- **列表不是数组**：随机访问（`LINDEX`）是 O(N)。位置查询请使用有序集合。
