# 练习 09：事务

## 你将学到
- 使用 `MULTI` / `EXEC` 原子性地批量执行命令
- 使用 `DISCARD` 取消事务
- 使用 `WATCH` 实现乐观锁
- 理解无回滚的限制

## 为什么重要
Redis 事务保证原子命令批处理——没有其他客户端会插入。对于转账等操作至关重要。`WATCH` 实现乐观并发控制。

## 常见陷阱
- 无回滚：事务中的运行时错误不会撤销之前的命令。
- WATCH 是连接作用域的，由 EXEC/DISCARD/断开连接清除。
- 中止时 EXEC 返回 None（或 redis-py 中抛出 WatchError）。
- MULTI 内的读取是延迟的——值在 EXEC 结果数组中返回。
