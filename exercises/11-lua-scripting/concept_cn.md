# 练习 11：Lua 脚本

## 你将学到
- 使用 `EVAL` 和 `EVALSHA` 执行 Lua 脚本
- 使用 `SCRIPT LOAD` 加载脚本以供缓存执行
- 实现原子检查并设置模式
- 理解 Lua 为什么是原子的：脚本会阻塞服务器

## 为什么重要
Lua 脚本是 Redis 最强大的原子性原语。与 MULTI/EXEC 不同，Lua 脚本可以原子地读取、计算和写入——非常适合限流器、库存扣减和分布式锁。脚本在服务器端运行，零网络往返。

## 常见陷阱
- **脚本阻塞服务器**：长时间运行的 Lua 脚本会阻塞所有其他客户端。保持脚本简短。
- **EVALSHA 需要回退**：如果脚本未被缓存，EVALSHA 返回 NOSCRIPT。始终回退到 EVAL。
- **无法访问外部 I/O**：Lua 沙箱没有网络、文件系统或时间访问。
