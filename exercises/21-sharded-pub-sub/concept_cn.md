# 练习 21：分片发布/订阅

## 你将学到
- 使用 `SSUBSCRIBE` 订阅分片频道
- 使用 `SPUBLISH` 向分片频道发布消息
- 使用 `SUNSUBSCRIBE` 取消订阅
- 理解分片与经典 Pub/Sub 的区别
- 知道何时使用每种模式

## 为什么重要

经典 Pub/Sub（练习 07）将每条消息广播到**每个集群节点**，无论该节点上是否存在订阅者。这在大规模下浪费带宽和 CPU。Redis 7.0 引入了分片 Pub/Sub，频道分布到集群分片——大大减少了跨节点流量。这是 Redis 集群部署的生产级 Pub/Sub 模式。

## 核心概念

### 经典 vs 分片

| 特性 | 经典 | 分片 |
|------|------|------|
| 订阅 | `SUBSCRIBE` | `SSUBSCRIBE` |
| 发布 | `PUBLISH` | `SPUBLISH` |
| 模式订阅 | `PSUBSCRIBE` | 不支持 |
| 频道分布 | 广播到所有节点 | 哈希到一个分片 |

## 常见陷阱

- **不支持模式订阅**：分片 Pub/Sub 没有 `PSUBSCRIBE`，使用经典 Pub/Sub 处理模式。
- **集群才有收益**：单节点 Redis 上分片 Pub/Sub 与经典无异。
- **需要 Redis 7.0+**：旧版本不支持分片 Pub/Sub 命令。
